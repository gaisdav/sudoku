# План разработки Sudoku

**Платформа:** мобильное приложение для **Android** и **iOS** на **Flutter**.

**Язык интерфейса:** по умолчанию **английский**. В будущем планируется локализация на 15–20 языков — закладывать структуру строк (например, через `flutter_localizations` / ARB или выбранное решение для i18n) с самого начала.

**Приоритеты:** большой упор на **плавность анимаций** и **приятный UI** — игра должна отзываться на действия пользователя плавно и выглядеть аккуратно на всех экранах.

---

## Этап 1: Функциональное ядро (MVP)

**Цель:** Работающая сетка 9×9, ввод цифр, проверка ошибок.

**Архитектура:**

- Провайдер состояния (Riverpod или BLoC): не пиши логику внутри виджетов. Создай `GameNotifier`, который хранит состояние игрового поля.

**Модель данных:**

- **SudokuCell:** класс с полями:
  - `value` (int)
  - `isOriginal` (bool) — цифра от генератора, нельзя стирать
  - `isHint` (bool) — цифра получена через подсказку (подсветить другим цветом, например синим)
  - `isWrong` (bool) — ошибка (подсветить красным)
- **SudokuBoard:** массив 9×9 из этих объектов.

Пример структуры клетки:

```dart
class SudokuCell {
  int value;
  bool isOriginal; // цифра от генератора (нельзя стирать)
  bool isHint;     // цифра получена через подсказку (подсветим синим)
  bool isWrong;    // ошибка (подсветим красным)
  // ...
}
```

**Функционал:**

- **Генерация:** использовать пакет **`sudoku_dart`** ([pub.dev/packages/sudoku_dart](https://pub.dev/packages/sudoku_dart)): генерация через `Sudoku.generate(Level)`, поля `.puzzle` и `.solution`. На этом этапе генерируй любой пазл при старте. Храни solution для подсказок.
- **Ввод цифр:** выбор клетки + панель кнопок 1–9 внизу.
- **Валидация:** клетка подсвечивается красным, если такая цифра уже есть в строке, столбце или квадрате 3×3.
- **Условие победы:** проверка, что все клетки заполнены верно.

**Подсказка (Hint) — подготовка логики («скрытая функция»):**

Кнопки в интерфейсе на этом этапе не нужна, но в `GameNotifier` уже должен быть метод для подсказки. Если не заложить `isHint` и логику в модель на первом этапе, потом придётся переписывать данные.

- Реализуй метод **`applyHint()`**, который:
  1. Находит целевую клетку: либо текущую выбранную игроком пустую, либо (умный вариант) если выбрана клетка с ошибкой — исправляет её.
  2. Берёт правильное число из «решённого» массива (solution от `sudoku_dart`).
  3. Записывает это число в поле и выставляет у клетки `isHint: true` (чтобы в UI потом подсветить её другим цветом).
- Имена в коде можно выбрать по вкусу (`fillHint()` / `applyHint()`); важно, чтобы один метод в логике по вызову подставлял правильную цифру и помечал клетку как подсказку.

## Этап 2: Красивый UI и анимации

**Цель:** Сделать игру приятной на ощупь («Juice»). Плавные анимации и продуманный интерфейс — ключевые требования проекта.

**Подготовка (уже заложена в Этапе 1):**

Поскольку логика отделена от UI, можно просто заменить `StatelessWidget` клетки на `StatefulWidget` или использовать Implicit Animations.

**Что добавить:**

- **AnimatedContainer:** для плавного изменения цвета фона выбранной клетки или строк/столбцов.
- **ScaleTransition:** чтобы цифра «впрыгивала» в клетку при нажатии.
- **CustomPainter:** для отрисовки границ блоков 3×3 (чтобы они были жирнее и красивее).
- **Тактильный отклик:** пакет `haptic_feedback` — лёгкая вибрация при каждом нажатии.
- **Иконка «Подсказка»:** нарисуй в UI кнопку-лампочку для подсказки (пока без монетизации — вызов `applyHint()` по нажатию можно добавить на следующем этапе или заглушить). Так кнопка не будет выглядеть «приклеенной» на этапе рекламы.

## Этап 3: Реклама (монетизация)

**Цель:** Интеграция AdMob без вреда для игрока.

**Архитектура:**

- Создай **AdService** — отдельный класс-синглтон. Не вызывай показ рекламы напрямую из экрана игры.

**Места для вставки:**

- **Banner:** в самом низу экрана.
- **Interstitial (межстраничная):** показывай только после завершения уровня (не во время игры!).
- **Rewarded (за вознаграждение):** кнопка «Подсказка» и/или «Удалить ошибки».

**Подсказка (Hint) — полноценная кнопка и монетизация:**

Подсказка — самый ценный ресурс в судоку. Если дать её бесплатно с первого этапа, игроки быстро пройдут игру и удалят приложение. Поэтому кнопку в интерфейс выводим здесь и привязываем к рекламе.

- **Логика:** игрок нажимает «Лампочку» → показывается диалог «Посмотреть видео, чтобы получить подсказку?» → после успешного просмотра Rewarded Video вызывается уже готовый метод **`applyHint()`** из Этапа 1.
- **Технически:** не вызывай показ рекламы из экрана игры напрямую. Через **AdService** запрашивай показ Rewarded; в колбэке успешного просмотра вызывай `applyHint()` в `GameNotifier`. Так кнопка просто триггерит рекламу, а логика подсказки остаётся в одном месте.

## Этап 4: Прогресс, сложность и статистика

**Цель:** Удержание игрока (Retention).

**Функционал:**

- **Выбор сложности:** при генерации через `sudoku_dart` передавай параметр `Level` (easy, medium, hard, expert).
- **Хранение данных:** используй пакет Isar или Hive (быстрые NoSQL базы).
  - Сохраняй текущую неоконченную игру (чтобы игрок мог вернуться).
  - Записывай время прохождения (таймер).
- **Статистика:** общее количество побед, лучший результат по времени для каждой сложности.

## Совет по масштабируемости

На Этапе 1 используй индексы от 0 до 80 для обращения к клеткам. Это упростит проверку квадратов 3×3.

Координаты клетки (в коде: `row = index ~/ 9`, `col = index % 9`):

- \( x = index \bmod 9 \) (столбец)
- \( y = index \div 9 \) (строка)

## Замечания, уточнения и вопросы к плану

- **Пакет для судоку:** выбран **`sudoku_dart`** ([pub.dev/packages/sudoku_dart](https://pub.dev/packages/sudoku_dart)): генерация `Sudoku.generate(Level.expert)`, доступ к `.puzzle` и `.solution`, уровни easy/medium/hard/expert. Альтернативы из топа: [sudoku_api](https://pub.dev/packages/sudoku_api) (Puzzle, board/solvedBoard), [fludoku](https://pub.dev/packages/fludoku) (generateSudokuPuzzle, findSolutions).
- **Пустая клетка:** явно зафиксировать в модели: пусто = `value == 0` (или `null` при использовании `int?`), чтобы единообразно обрабатывать ввод и отрисовку.
- **Состояние выбора:** в `GameNotifier` хранить `selectedCellIndex` (int? или 0–80), чтобы была «текущая выбранная клетка» для ввода и для `applyHint()`.
- **Когда пересчитывать `isWrong`:** пересчитывать при любом изменении поля (после ввода/удаления/подсказки), чтобы подсветка ошибок была актуальной.
- **Иммутабельность:** при изменении поля создавать новый список/копию доски, а не мутировать клетки на месте — так Riverpod корректно обновит UI.
- **Кнопка «Подсказка»:** на Этапе 2 кнопка уже в UI (лампочка); на Этапе 3 к ней привязывается реклама. Уточнить: на Этапе 2 по нажатию вызывать `applyHint()` без рекламы (для теста) или оставить заглушку.
- **«Удалить ошибки»:** в Этапе 3 упомянуто, но логика не расписана. Решить: удалять только ячейки с `isWrong == true` (оставляя `isOriginal` и `isHint`), и реализовать метод в `GameNotifier` (например, `clearWrongCells()`).
- **Победа в MVP:** явно указать: при выполнении условия победы показывать диалог «Победа!» (детали экрана победы и навигации — на усмотрение или в Этапе 4).
- **Сложность (Этап 4):** у выбранного пакета проверить, есть ли параметр сложности (Easy/Medium/Hard); если нет — задавать сложность числом открытых клеток или уровнем удаления цифр.
