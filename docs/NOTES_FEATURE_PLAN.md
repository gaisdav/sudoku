# План: режим Notes (черновые числа)

## Ваше видение (кратко)

- Включили кнопку Notes → можно ставить черновые числа в пустые ячейки.
- Разрешены только те цифры, которые ещё могут стоять в этой ячейке; заведомо неверные подсвечиваются и **не ставятся**.
- В режиме Notes **отключить** счётчик оставшихся чисел на клавиатуре (1–9).
- Основные числа после ввода остаются и при выключенном Notes.
- Черновые числа **пропадают** в ячейке, как только туда ставится основное число (с выключенным Notes).
- Черновые цифры рисуются **мелким шрифтом** внутри ячейки в сетке 3×3: позиция = цифра (1–9).

---

## 1. Модель данных

### 1.1 Ячейка

- В **SudokuCell** добавить поле черновых цифр, например:
  - `Set<int> notes` (пустой по умолчанию), цифры 1–9.
- Либо отдельная структура на уровне поля 9×9: `List<Set<int>> notesPerCell` (81 элемент) в **GameState** — тогда ячейка остаётся без изменений, а черновики хранятся в состоянии игры.  
  **Рекомендация:** хранить в **GameState** (отдельный список/мапа по индексу 0..80), чтобы не менять контракт ячейки и проще сериализовать.

### 1.2 Состояние игры

- В **GameState** добавить:
  - `bool isNotesMode` — режим Notes включён/выключен.
  - `List<Set<int>> cellNotes` — для каждой из 81 ячейки множество цифр 1–9 (черновики). Для заполненных ячеек можно хранить пустой set и не показывать черновики.
- При установке **основного** числа в ячейку (режим не Notes): для этой ячейки очищать `cellNotes[index]` (черновые исчезают сразу).

---

## 2. Валидация черновых чисел

- **Кандидаты для ячейки:** цифры 1–9, которые ещё не противоречат текущему заполнению (нет такой же цифры в той же строке, столбце, блоке 3×3).
- В режиме Notes:
  - **Добавить черновик N** в ячейку: разрешать только если N входит в кандидаты этой ячейки. Если не входит — не добавлять, подсветить кнопку N на клавиатуре как «недоступную» (например, красным/серым) и не ставить число в ячейку.
  - **Удалить черновик:** всегда можно (повторное нажатие N или отдельное действие).
- Кандидаты пересчитывать от текущего состояния доски (только `cells`, без учёта черновиков).

---

## 3. Расположение черновых цифр в ячейке

Стандартная сетка 3×3 внутри ячейки:

- Позиция 1 — верх-лево, 2 — верх-центр, 3 — верх-право  
- 4 — середина-лево, 5 — центр, 6 — середина-право  
- 7 — низ-лево, 8 — низ-центр, 9 — низ-право  

Черновые цифры рисуются **мелким шрифтом** в соответствующих позициях (по одной цифре в каждой позиции, не несколько в одной клетке).  
Уточнение: у вас было «слева сверху можно 1, сверху посередине можно 2, слева сверху можно 3» — для 3 логично **справа сверху** (позиция 3), иначе 1 и 3 совпадают. В плане принята схема выше.

---

## 4. Клавиатура (Number Pad) в режиме Notes

- Когда **Notes включён**:
  - **Счётчик оставшихся чисел** (сколько раз ещё можно поставить 1, 2, … 9) **не показывать** — выключить отображение `remaining` для кнопок 1–9.
  - Кнопки 1–9: действие = добавить/убрать черновик в выбранной ячейке (toggle). Если цифра не в кандидатах — не добавлять, подсветить кнопку.
  - Кнопка **Clear (×)**: в режиме Notes — очистить все черновики в выбранной ячейке (или оставить «очистить основное число» только при выключенном Notes; см. ниже).
- Когда **Notes выключен** — поведение как сейчас: 1–9 ставят основное число, Clear очищает ячейку; счётчик оставшихся показывается на Easy/Medium.

---

## 5. Поведение при переключении Notes и при вводе

- **Включили Notes** — только добавление/удаление черновиков в пустых (и выбранной) ячейках; основное число не ставится.
- **Выключили Notes** — ввод 1–9 снова ставит основное число. Черновики на доске **остаются** (не стираются при выключении режима).
- **Поставили основное число в ячейку при выключенном Notes** — в этой ячейке все черновики для неё **сразу удаляются** (только в этой ячейке).

---

## 6. Кнопка Notes в UI

- В панели под сеткой (рядом с Hint) **включить** кнопку Notes (сейчас скрыта).
- Иконка, например `Icons.edit_note`, подпись «Notes».
- Toggle: нажатие включает/выключает `isNotesMode`. Визуально показывать активное состояние (другая заливка/обводка, когда режим включён).

---

## 7. Сохранение и загрузка (GameStorage)

- В сохраняемую игру добавить:
  - `isNotesMode` (bool),
  - `cellNotes` — массив из 81 элемента; каждый элемент — список/множество цифр 1–9 (сериализовать как список чисел, например `[[1,3],[],[2,5],...]`).
- При загрузке игры восстанавливать режим Notes и все черновики по ячейкам.

---

## 8. Ответы (зафиксировано)

| Вопрос | Ответ |
|--------|--------|
| Кнопка **Clear** в режиме Notes | **Да** — очищает только черновики в выбранной ячейке. |
| Подсветка при недопустимом черновике | **Заметное красное мигание** ячейки, в которую хотели поставить черновик, **и ячейки с исходным числом**, с которым конфликт. Конфликт только с цифрами **при инициализации игры** (isOriginal). Цифры, поставленные игроком, не считаются конфликтом — черновик с таким числом разрешён. На клавиатуре — красное мигание кнопки цифры. |

---

## 9. Порядок реализации (предложение)

1. **Модель и провайдер:** в GameState добавить `isNotesMode`, `cellNotes`; в провайдере — переключение режима, добавление/удаление черновика с проверкой кандидатов, очистка черновиков при установке основного числа; обновить сохранение/загрузку.
2. **Ячейка:** в `SudokuCellWidget` (или в виджете сетки) рисовать черновики мелким шрифтом в сетке 3×3 по позициям 1–9.
3. **Клавиатура:** при `isNotesMode` скрыть счётчики, кнопки 1–9 — toggle черновика с валидацией и подсветкой; Clear — очистка черновиков выбранной ячейки.
4. **Кнопка Notes:** показать в UI, привязать к `isNotesMode`.
5. **Тесты:** сохранение/загрузка с черновиками, очистка черновиков при вводе основного числа, запрет недопустимых черновиков.

---

## 10. Итог

- Режим Notes включается кнопкой; черновые числа только в пустых ячейках, только допустимые цифры; недопустимые не ставятся, кнопка на клавиатуре подсвечивается.
- Счётчик оставшихся чисел при Notes отключён.
- Основные числа сохраняются при выключении Notes; черновики в ячейке исчезают только при вводе в неё основного числа (с выключенным Notes).
- Черновики отображаются мелким шрифтом в сетке 3×3 (позиции 1–9), сохраняются в игре вместе с доской.

Если что-то из этого расходится с вашей задумкой (например, поведение Clear или подсветка недопустимой цифры), можно точечно поменять только эти пункты и потом переходить к реализации.
